## 14. DESIGN DIRECTION (Premium — 2026 Standard)

### Reference Sites

Study and implement patterns from:
- **Linear** — Dark mode, precision typography, animation as product demo
- **Vercel** — High contrast, editorial layout, speed as aesthetic
- **Stripe** — Gradient accents, documentation-as-surface, code samples
- **Ramp** — Clean dashboard-style layouts
- **Raycast** — Minimal, functional, keyboard-first
- **Resend** — Developer-focused, technical credibility
- **Dub** — Modern, clean, conversion-focused

### Design Principles

```
1. HIGH CONTRAST
   Black on white, white on black.
   Nothing muddy in between.
   Eye knows exactly where to go.

2. GENEROUS WHITESPACE
   More space than seems necessary.
   Eye relaxes, content does the job.
   Dense layouts = eye fatigue.

3. MONOCHROME + ONE ACCENT
   Black, white, gray base.
   One accent color (yellow-500 for Reverbex).
   Used sparingly — never all three at once.
   Accent appears at most once per scroll viewport.

4. SHARP TYPOGRAPHY
   Tight, geometric, slightly cold.
   No rounded, friendly, approachable fonts.
   Communicates: "built by engineers who care about craft"

5. DARK THEME
   Default surface is dark theme.
   Accent appears at hero CTA and nowhere else.
   No gradient backgrounds.
   No rounded corners over 8px.

6. MOTION AS PROOF
   Animations demonstrate product value.
   Not decorative — functional.
   Linear shows speed through animation.
   Reverbex shows performance through animation.

7. MASSIVE TYPOGRAPHY
   Hero headlines at 6-8rem.
   Section headers at 3-4rem.
   Body at 1-1.25rem.
   Fluid scaling with clamp().

8. CODE AS CREDENTIAL
   Show actual code snippets.
   Terminal-style interfaces.
   Command-line outputs as social proof.
   Performance metrics as hero visual.
```

### Key Architecture Decisions

#### 1. Server Components by Default

```tsx

// Can fetch data directly

export default async function ServicePage({ params }: { params: Promise<{ slug: string }> }) {

    </div>
```


### GSAP + Lenis Integration (Production)

```tsx
// components/providers/LenisProvider.tsx
"use client";

import { ReactLenis, useLenis } from "lenis/react";
import { useEffect } from "react";
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";

gsap.registerPlugin(ScrollTrigger);

function LenisGSAPConnector() {
  const lenis = useLenis();
  
  useEffect(() => {
    if (!lenis) return;
    
    // Synchronize Lenis with ScrollTrigger
    lenis.on("scroll", ScrollTrigger.update);
    
    // Drive Lenis from GSAP ticker (single RAF loop)
    const update = (time: number) => {
      lenis.raf(time * 1000); // GSAP gives seconds, Lenis expects milliseconds
    };
    
    gsap.ticker.add(update);
    gsap.ticker.lagSmoothing(0);
    
    return () => {
      lenis.off("scroll", ScrollTrigger.update);
      gsap.ticker.remove(update);
    };
  }, [lenis]);
  
  return null;
}

export default function LenisProvider({ children }: { children: React.ReactNode }) {
  return (
    <ReactLenis
      root
      options={{
        lerp: 0.1,
        duration: 1.2,
        smoothWheel: true,
        wheelMultiplier: 1,
        touchMultiplier: 2,
        syncTouch: false, // Better mobile performance
        autoRaf: false,   // We're driving from GSAP ticker
      }}
    >
      <LenisGSAPConnector />
      {children}
    </ReactLenis>
  );
}
```

#### 8. Magnetic Elements

Buttons and interactive elements that pull toward the cursor when nearby. Creates tactile, alive feel.

```tsx
// Magnetic button with spring physics
function MagneticButton({ children }) {
  const ref = useRef(null);
  
  useEffect(() => {
    const el = ref.current;
    const handleMouseMove = (e) => {
      const rect = el.getBoundingClientRect();
      const x = e.clientX - rect.left - rect.width / 2;
      const y = e.clientY - rect.top - rect.height / 2;
      
      gsap.to(el, {
        x: x * 0.3,
        y: y * 0.3,
        duration: 0.3,
        ease: 'power2.out',
      });
    };
    
    const handleMouseLeave = () => {
      gsap.to(el, { x: 0, y: 0, duration: 0.5, ease: 'elastic.out(1, 0.3)' });
    };
    
    el.addEventListener('mousemove', handleMouseMove);
    el.addEventListener('mouseleave', handleMouseLeave);
    return () => {
      el.removeEventListener('mousemove', handleMouseMove);
      el.removeEventListener('mouseleave', handleMouseLeave);
    };
  }, []);
  
  return <button ref={ref}>{children}</button>;
}
```

**Use for Reverbex:** WhatsApp CTA button, social media icons, navigation links

```tsx
// Dual-row marquee with scroll velocity modulation
function Marquee({ items, direction = 'left' }) {
  const [velocity, setVelocity] = useState(0);
  
  useEffect(() => {
    const lenis = useLenis();
    lenis.on('scroll', ({ velocity }) => setVelocity(velocity));

        transition={{

        }}
      >
        {[...items, ...items].map((item, i) => (
          <span key={i} className="text-6xl font-bold text-white/10">
            {item}
          </span>
        ))}
      </motion.div>
    </div>
  );
}
```

#### 10. 3D Card Tilt with Parallax

Cards that tilt toward the cursor with layered depth. Apple TV-style parallax.

```tsx
function TiltCard({ children }) {
  const ref = useRef(null);
  
  useGSAP(() => {
    const el = ref.current;
    el.addEventListener('mousemove', (e) => {
      const rect = el.getBoundingClientRect();
      const x = (e.clientX - rect.left) / rect.width - 0.5;
      const y = (e.clientY - rect.top) / rect.height - 0.5;
      
      gsap.to(el, {
        rotateY: x * 20,
        rotateX: -y * 20,
        transformPerspective: 1000,
        duration: 0.3,
        ease: 'power2.out',
      });
    });
    
    el.addEventListener('mouseleave', () => {
      gsap.to(el, {
        rotateY: 0,
        rotateX: 0,
        duration: 0.5,
        ease: 'elastic.out(1, 0.5)',
      });
    });
  }, { scope: ref });
  
  return (
    <div ref={ref} className="relative" style={{ transformStyle: 'preserve-3d' }}>
      {children}
    </div>
  );
}
```

```tsx
function ScrambleText({ text, className }) {
  const ref = useRef(null);
  const chars = "!@#$%^&*()_+{}|:<>?-=[]\\;',./~`";
  
  useEffect(() => {
    const el = ref.current;
    let iterations = 0;
    
    const interval = setInterval(() => {
      el.innerText = text
        .split("")
        .map((char, i) => {
          if (i < iterations) return text[i];
          return chars[Math.floor(Math.random() * chars.length)];
        })
        .join("");
      
      if (iterations >= text.length) clearInterval(interval);
      iterations += 1 / 3;
    }, 30);
    
    return () => clearInterval(interval);
  }, [text]);
  
  return <span ref={ref} className={className} />;
}
```

#### Magnetic Text (Per-Character Pull)
Each letter physically translates toward cursor with spring physics.

```tsx
function MagneticText({ text, radius = 100, strength = 0.3 }) {
  const containerRef = useRef(null);
  
  useEffect(() => {
    const container = containerRef.current;
    const chars = container.querySelectorAll(".magnetic-char");
    
    const handleMouseMove = (e) => {
      chars.forEach((char) => {
        const rect = char.getBoundingClientRect();
        const charX = rect.left + rect.width / 2;
        const charY = rect.top + rect.height / 2;
        const distX = e.clientX - charX;
        const distY = e.clientY - charY;
        const dist = Math.sqrt(distX * distX + distY * distY);
        
        if (dist < radius) {
          const pull = (1 - dist / radius) * strength;
          gsap.to(char, {
            x: distX * pull,
            y: distY * pull,
            duration: 0.3,
            ease: "power2.out",
          });
        } else {
          gsap.to(char, { x: 0, y: 0, duration: 0.6, ease: "elastic.out(1, 0.3)" });
        }
      });
    };
    
    container.addEventListener("mousemove", handleMouseMove);
    return () => container.removeEventListener("mousemove", handleMouseMove);
  }, [radius, strength]);
  
  return (
    <div ref={containerRef} className="flex">
      {text.split("").map((char, i) => (
        <span key={i} className="magnetic-char inline-block cursor-none">
          {char === " " ? "\u00A0" : char}
        </span>
      ))}
    </div>
  );
}
```

```tsx
    let lastX = 0, lastY = 0, distance = 0;
    
    const handleMouseMove = (e) => {
      const dx = e.clientX - lastX;
      const dy = e.clientY - lastY;
      distance += Math.abs(dx) + Math.abs(dy);
      
      if (distance > 100) {
        distance = 0;
        setParticles(prev => [...prev.slice(-10), {
          id: Date.now(),
          x: e.clientX,
          y: e.clientY,
          dx: dx * 0.5,
        }]);
      }
      
      lastX = e.clientX;
      lastY = e.clientY;
    };
    
    window.addEventListener("mousemove", handleMouseMove);
    return () => window.removeEventListener("mousemove", handleMouseMove);
  }, []);
  
  useEffect(() => {
    particles.forEach(p => {
      gsap.to(`#particle-${p.id}`, {
        y: window.innerHeight,
        x: `+=${p.dx * 5}`,
        rotation: Math.random() * 360,
        opacity: 0,
        duration: 1.5,
        ease: "bounce.out",
        onComplete: () => {
          setParticles(prev => prev.filter(pp => pp.id !== p.id));
        },
      });
    });
  }, [particles]);
  
  return (
    <>
      {particles.map(p => (
        <div
          key={p.id}
          id={`particle-${p.id}`}
          className="fixed w-8 h-8 bg-white/20 rounded pointer-events-none z-50"
          style={{ left: p.x, top: p.y, transform: "translate(-50%, -50%)" }}
```

```tsx
function ParticleField({ count = 50 }) {
  const canvasRef = useRef(null);
  
  useEffect(() => {
    const canvas = canvasRef.current;
    const ctx = canvas.getContext("2d");
    let animationId;
    
    canvas.width = window.innerWidth;
    canvas.height = window.innerHeight;
    
    const particles = Array.from({ length: count }, () => ({
      x: Math.random() * canvas.width,
      y: Math.random() * canvas.height,
      vx: (Math.random() - 0.5) * 0.5,
      vy: (Math.random() - 0.5) * 0.5,
      size: Math.random() * 2 + 0.5,
      opacity: Math.random() * 0.3 + 0.1,
    }));
    
    function animate() {
      ctx.clearRect(0, 0, canvas.width, canvas.height);
      
      particles.forEach(p => {
        p.x += p.vx;
        p.y += p.vy;
        
        if (p.x < 0) p.x = canvas.width;
        if (p.x > canvas.width) p.x = 0;
        if (p.y < 0) p.y = canvas.height;
        if (p.y > canvas.height) p.y = 0;
        
        ctx.beginPath();
        ctx.arc(p.x, p.y, p.size, 0, Math.PI * 2);
        ctx.fillStyle = `rgba(255, 255, 255, ${p.opacity})`;
        ctx.fill();
      });
      
      animationId = requestAnimationFrame(animate);
    }
    
    animate();
    return () => cancelAnimationFrame(animationId);
  }, [count]);
  
  return (
    <canvas 
      ref={canvasRef} 
      className="fixed inset-0 pointer-events-none z-0" 
    />
  );
}
```

### J. Scroll Pauses (Pacing Control)

The site "holds" you. Section locks. Animation happens. Then unlocks. Creates pacing and drama.

```tsx
function ScrollPause({ children, animation }) {
  const ref = useRef(null);
  
  useGSAP(() => {
    // Pin the section
    ScrollTrigger.create({
      trigger: ref.current,
      start: "top top",
      end: "+=100%", // Hold for 1 extra viewport
      pin: true,
      scrub: false,
      onEnter: () => {
        // Play animation while pinned
        animation();
      },
    });
  }, { scope: ref });
  
  return <div ref={ref}>{children}</div>;
}
```

### K. Living Numbers (Animated Counters)

Static `500+` is boring. Counter that grows as story progresses is engaging.

```tsx
function LivingCounter({ target, duration = 2000, prefix = "", suffix = "+" }) {
  const [count, setCount] = useState(0);
  const ref = useRef(null);
  const hasAnimated = useRef(false);
  
  useGSAP(() => {
    ScrollTrigger.create({
      trigger: ref.current,
      start: "top 80%",
      onEnter: () => {
        if (hasAnimated.current) return;
        hasAnimated.current = true;
        
        const start = Date.now();
        const animate = () => {
          const elapsed = Date.now() - start;
          const progress = Math.min(elapsed / duration, 1);
          // Ease out cubic
          const eased = 1 - Math.pow(1 - progress, 3);
          setCount(Math.floor(eased * target));
          if (progress < 1) requestAnimationFrame(animate);
        };
        animate();
      },
    });
  }, [target, duration]);
  
  return (
    <span ref={ref} className="tabular-nums">
      {prefix}{count.toLocaleString()}{suffix}
    </span>
  );
}
```

### M. Prompting Rules for AI Tools (v0, Cursor, Claude)

#### The 8 Meta-Rules That Make Any Design Prompt Better

1. **Assign a role.** "You are a senior product designer." — measurably raises output quality.
2. **Specify the stack.** Name Tailwind, shadcn/ui, Radix. Without it, AI guesses.
3. **Give a spacing scale.** "Use the 8px spacing scale" prevents random-gutter AI look.
4. **Demand real content.** No lorem ipsum. Give actual headlines, actual stats, actual copy.
5. **Ask for all interactive states.** Hover, focus, active, disabled, loading, empty.
6. **Constrain the palette.** "#6366F1 as primary, neutral grays elsewhere" beats "make it modern."
7. **Request variants in one artifact.** "Show A, B, C with a toggle" — compare without round-trips.
8. **End with a rationale request.** "Explain your layout choices in 3 bullets" — exposes reasoning.

#### Design System Block (Add to Every Prompt)

```
DESIGN SYSTEM:
- Background: #050505 (near-black)
- Text: #F5F5F0 (warm white)
- Accent: #EAB308 (Reverbex yellow)
- Font Headings: Syne (variable weight)
- Font Body: DM Sans (300-600)
- Spacing: 8px base unit
- Border radius: 0px (sharp) or 12px (cards)
- Shadows: Layered (0 1px 2px, 0 4px 8px, 0 8px 16px)
- No gradients. No glassmorphism. No neon.
- Mood: Editorial, premium, confident. Like Linear meets a design magazine.
```

#### Cursor Rules (.cursorrules)

```
- Always import from @/components/ui/ (shadcn)
- Never create new button, card, input components — use existing
- Use Tailwind classes, not inline styles
- Animate only transform and opacity (GPU-composited)
- All animations must respect prefers-reduced-motion
- Use CSS scroll-driven animations before GSAP
- Server Components by default — add "use client" only when needed
- Use next/image for all images
- Use next/font for all fonts
```

#### Common Prompting Mistakes

| Mistake | Why It Fails | Fix |
|---------|-------------|-----|
| "Make it modern" | AI guesses what modern means | "#0A0A0A bg, sharp corners, Syne font" |
| "Add animations" | Decorative junk everywhere | "Fade-in on scroll, magnetic button, counter on proof" |
| "Full page in one prompt" | Diff too large to review | One component per prompt |
| "Refactor everything" | Breaks working code | "Edit components/Hero.tsx only" |
| No constraints given | Accessibility dropped silently | Always state: responsive, keyboard-accessible, dark mode |
| Accepting output without review | Focus order, labels drift | Review every generation |

### N. The 12 Don'ts (Lessons from Award-Winning Designers)

#### 1. DON'T center everything
Centered heroes are the default. Break the pattern. Left-aligned text is more editorial, more premium, more intentional.

#### 2. DON'T use placeholder content
Real headlines. Real stats. Real case study names. AI-generated filler screams "template."

#### 3. DON'T animate everything
If it doesn't guide attention or confirm an action, remove it. Decorative motion = code bloat + slower page.

#### 4. DON'T use glassmorphism on plain backgrounds
Glassmorphism needs rich backgrounds (images, gradients, video). Over white/black, it's invisible.

#### 5. DON'T load 5 fonts
Two max (one heading, one body). One is better. Each weight = separate HTTP request.

#### 6. DON'T build for desktop first
60%+ of traffic is mobile. Design mobile first. Test on real devices, not DevTools simulation.

#### 7. DON'T use stock images
Real screenshots. Real dashboards. Real WhatsApp conversations. Real analytics. Authenticity > beauty.

#### 8. DON'T add particles/WebGL for decoration
If it doesn't communicate something, it's dead weight. Particles don't help conversion.

#### 9. DON'T copy Linear/Vercel/Stripe literally
Study their principles (contrast, whitespace, monochrome + accent). Don't clone their layouts.

#### 10. DON'T skip accessibility
`prefers-reduced-motion`. Focus rings. ARIA labels. Alt text. This is not optional.

#### 11. DON'T accept AI output as final
Every generated component needs review: focus order, labels, edge states, TypeScript types, security.

#### 12. DON'T over-specify then under-iterate
Give enough context for the first draft. Then iterate in small, focused steps. One change per prompt.

### C. AI Anti-Patterns — What NOT to Do

#### The 15 AI Design Anti-Patterns

| # | Anti-Pattern | Why It Fails | Fix |
|---|-------------|-------------|-----|
| 1 | **"Make it modern"** | AI guesses what modern means. Default = purple gradients + Inter font + cards | "#0A0A0A bg, sharp corners, Syne font, editorial layout" |
| 2 | **Purple-to-blue gradients** | Most overused AI color palette. Appears in 90% of generated UI | Use brand-specific colors. Reverbex: #EAB308 yellow on dark |
| 3 | **Inter everywhere** | "The Comic Sans of AI" — default font that signals no thought went into typography | Name specific fonts. "Syne for headings, DM Sans for body" |
| 4 | **Glassmorphism on everything** | AI learned blur = premium. Applies it decoratively, not purposefully | Reserve for hero overlays only. Never on text-heavy sections |
| 5 | **Card nesting** | Everything wrapped in cards. Cards inside cards. Three levels of containers | Flatten hierarchy. Use whitespace, not borders, for separation |
| 6 | **3-column identical feature grids** | Most common AI layout. Zero visual hierarchy | Use bento grid, expand-on-hover, or editorial asymmetric layout |
| 7 | **Center-aligned everything** | Centered heroes, centered text, centered CTAs. No editorial feel | Left-align text. Use asymmetric grids. Center only for hero numbers |
| 8 | **Stock images / lorem ipsum** | Screams "template." No authenticity | Real screenshots. Real stats. Real case study names |
| 9 | **Decorative animations** | Fade-ins on load that add no information. User waits longer to read | Animate only to guide attention or confirm action |
| 10 | **No negative constraints** | AI needs to know what NOT to do. Without don'ts, it defaults to common patterns | Always include a DON'T section in prompts |
| 11 | **Whole app in one prompt** | Diff too large to review. AI makes wrong architectural decisions | One component per prompt. Iterate in small steps |
| 12 | **Accepting first output** | Generated UI drifts on focus order, labels, edge states | Review every generation. Run accessibility checks |
| 13 | **Gradient text on metrics** | Overused for "impact." Looks like every other AI dashboard | Use plain text with weight/size hierarchy |
| 14 | **Generic hero copy** | "Build the future with AI-powered..." — meaningless filler | Real headline. Real value prop. Max 6 words |
| 15 | **No component states** | Only default state shown. No hover, focus, active, disabled, loading, empty | Always request all 6 interactive states |

### E. Prompt Engineering Rules for Each Tool

#### v0 (Vercel) — Component-Level

```
Good prompt structure:
1. Component type (hero, card, form, navigation)
2. Visual intent (layout, hierarchy, mood)
3. Constraints (stack: Next.js + Tailwind + shadcn)
4. States (hover, focus, disabled, loading)
5. Real content (actual headlines, actual stats)

Bad: "Make me a landing page"
Good: "Hero section for Reverbex Technologies. Left-aligned. 
      Syne font 80px weight 900. Dark bg #050505. Yellow accent #EAB308. 
      One sentence: 'Websites. Ads. Automation. Built To Grow Businesses.'
      Single CTA button: 'Get Started'. Mobile responsive."
```

#### Cursor — Codebase-Aware

```
Good prompt structure:
1. Specific file to edit (components/Hero.tsx)
2. What to change (specific, bounded)
3. What NOT to change (scope lock)
4. Reference existing patterns ("use Button from @/components/ui")

Bad: "Improve the homepage"
Good: "Edit components/Hero.tsx only. Change the headline to 
      'Websites. Ads. Automation.' Keep all other components untouched. 
      Use the existing Button component from @/components/ui/button."
```

#### Claude Design — System-Level

```
Good prompt structure:
1. Assign role: "You are a senior product designer"
2. Specify stack: "Tailwind, shadcn/ui, Radix"
3. Give spacing scale: "8px base unit"
4. Demand real content: No lorem ipsum
5. Constrain palette: Exact hex values
6. Request all states: Hover, focus, active, disabled, loading, empty
7. End with rationale: "Explain your layout choices in 3 bullets"
```

### F. The 69 Design Rules (from StyleSeed)

These are brand-agnostic rules that teach LLMs design judgment:

#### Color Discipline
1. Use semantic tokens, not raw hex values
2. 60-30-10 rule: 60% dominant, 30% secondary, 10% accent
3. Accent color only for CTAs, active states, links — never decoration
4. Status colors are reserved — never repurpose for decoration
5. Dark mode uses surface elevation, not just inverted colors

#### Spatial Rhythm
6. 8pt grid — every margin, padding, gap lands on 8, 16, 24, 32...
7. No arbitrary pixel values
8. Section isolation: surface shift, whitespace, divider, border, or color fill
9. Never stack two same-surface sections without separation
10. Generous whitespace — premium = more space, not more elements

#### Information Hierarchy
11. One primary action per view
12. Text flows through three levels: primary → muted → faint
13. Headlines: tight tracking (-0.02em to -0.03em)
14. Body text: relaxed tracking (0), line-height 1.5-1.7
15. Accent font weight for emphasis, not size

#### Shadow & Elevation
16. Layered shadows, not single drop shadows
17. `box-shadow: 0 1px 2px, 0 4px 8px, 0 8px 16px`
18. Shadow intensity correlates with elevation
19. No shadows on flat elements
20. Border + shadow = redundant. Pick one.

#### Component Variance
21. Every button needs 6 states: default, hover, focus, active, disabled, loading
22. Cards use border OR shadow, never both
23. Form inputs need: default, focus, error, disabled, success
24. Loading states use skeleton, not spinners
25. Empty states need illustration + action, not just text

#### Motion & Feedback
26. Animate only transform and opacity (GPU-composited)
27. Never animate width, height, top, left, margin, padding
28. Duration: 200ms hover, 300ms transition, 500ms page
29. Easing: ease-out for entrances, ease-in for exits
30. Respect prefers-reduced-motion: reduce

### G. The Diagnostic Loop (Self-Verification)

After every AI generation, run this check:

```
1. Generate 3 screens (home, service, case study)
2. Identify where output drifts from design system
3. Add the missing constraint to DESIGN.md or prompt
4. Regenerate
5. Repeat until output is consistent
```

**What to check:**
- □ Are colors from the token system? (no random hex)
- □ Is typography on the defined scale? (no arbitrary sizes)
- □ Does spacing follow the 8pt grid? (no random padding)
- □ Are there 6 states for every interactive element?
- □ Is the layout editorial, not centered-card-grid?
- □ Does it pass WCAG AA contrast?
- □ Is prefers-reduced-motion handled?
- □ Are there any anti-patterns present?
- □ Does it feel like Reverbex, not "AI default"?
