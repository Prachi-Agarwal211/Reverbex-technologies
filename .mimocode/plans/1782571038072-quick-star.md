# Reverbex Homepage Redesign Plan — Cinematic Scroll Experience

## Scope
Redesign the homepage (keep HeroVideo as-is) with cinematic, image-driven, scroll-linked animations. Focus: **Services section** (primary), plus upgrades to FeaturedArchitectures, WhyReverbex, Methodology, ReverbexBond, FAQ, and Contact sections.

---

## 1. SERVICES SECTION — 3D Card Flip Split (PRIMARY TARGET)

### Concept
The 12 services start as 3 panoramic "sliced" images (each showing a collage of the services). On scroll:
1. The 3 images expand from a narrow slit to fill the viewport width
2. They split apart into separate cards with gap + border-radius
3. Each card flips 180° (Y-axis) to reveal the service name, description, and tags on the back
4. Cards fan out slightly during the flip for dramatic effect

### Approach: Group 12 services into 3 "rounds" of 4 cards each
- **Round 1**: Services 01-04 (Websites, E-Commerce, Mobile Apps, Meta Ads)
- **Round 2**: Services 05-08 (Google Ads, Lead Gen, ERP, WhatsApp)
- **Round 3**: Services 09-12 (AI Solutions, SEO, Branding, Rebranding)

Each round gets one scroll-pinned sequence (~4 viewport heights of scroll). Between rounds, a brief transition section provides breathing room.

### Files to Create/Modify

#### A. `components/ServicesFlipSection.tsx` (NEW — replaces StickyServices)
```
- Imports Flip from 'gsap/Flip' (register plugin)
- Imports ScrollTrigger (already registered in SmoothScroll)
- Uses useGSAP hook with gsap.context for cleanup
- 3 .flip-round containers, each with 4 .flip-card elements
- Each card has .card-front (atmospheric image) and .card-back (service info)
- ScrollTrigger pins each round, scrub: 1
- Uses onUpdate with progress-based state machine:
  - 0.0-0.25: Cards expand from slit (clipPath inset animation)
  - 0.25-0.50: Cards separate (gap increases, border-radius appears)
  - 0.50-0.85: Cards flip (rotateY 0→180°, staggered with slight rotateZ fan)
  - 0.85-1.00: Hold state
```

#### B. `components/ServiceCardBack.tsx` (NEW)
```
- Renders service name (Cormorant italic), description, tech tags
- Backface of the flip card
- Uses existing servicesData for content
```

#### C. `components/ServiceCardFront.tsx` (NEW)
```
- Full-bleed atmospheric image from Unsplash
- Service number overlay (top-left, monospace)
- Gold accent line at bottom
- Image treatment: saturate(0.85) contrast(1.05) brightness(0.95)
```

#### D. CSS additions to `globals.css`
```css
/* 3D flip card system */
.flip-card {
  perspective: 1000px;
  transform-style: preserve-3d;
}
.flip-card-inner {
  position: relative;
  width: 100%;
  height: 100%;
  transform-style: preserve-3d;
  backface-visibility: hidden;
}
.flip-card-front, .flip-card-back {
  position: absolute;
  inset: 0;
  backface-visibility: hidden;
}
.flip-card-back {
  transform: rotateY(180deg);
}

/* Slit expansion */
.flip-slit {
  clip-path: polygon(49.5% 0%, 50.5% 0%, 50.5% 100%, 49.5% 100%);
}
```

### Unsplash Image Strategy
Search terms for each service group:
- **Round 1** (Web/E-comm/Mobile/Ads): "circuit board macro", "fiber optic close up", "server room dark", "digital interface"
- **Round 2** (Ads/Gen/ERP/WhatsApp): "code editor dark", "data visualization", "dashboard dark", "network connections"
- **Round 3** (AI/SEO/Branding): "artificial intelligence abstract", "search algorithm", "brand identity design", "creative process"

All images must be dark-toned, high-contrast, desaturated to match the gold accent.

### Mobile Fallback (≤768px)
- Disable 3D flip (no rotateY)
- Instead: cards slide up from bottom with clipPath reveal
- Each card shows full content without flip interaction
- Reduced scroll distance (2 viewport heights per round instead of 4)
- Images become background with gradient overlay

## KEY TECHNICAL NOTES

### GSAP Flip Plugin
- **Not included in current package.json** — need to import from gsap core (Flip is bundled with gsap@3.14.2+)
- Register: `gsap.registerPlugin(Flip)` in the component
- Usage: `Flip.getState(elements)` → make DOM changes → `Flip.from(state, { duration, ease })`

### ScrollTrigger Pin Math for 3D Flip
```
Total scroll per round = window.innerHeight * 3 (3 extra viewports of scroll)
progress 0.00-0.25: clipPath expansion (slit → full width)
progress 0.25-0.40: gap + border-radius appear
progress 0.40-0.75: rotateY 0→180° with stagger
progress 0.75-1.00: hold
```

### Mobile Detection
```typescript
const isMobile = window.matchMedia('(max-width: 768px)').matches
// Skip 3D flip on mobile, use clipPath reveal instead
```

### Image Loading
- Use Next.js `<Image>` with `priority` for above-fold cards
- Use `loading="lazy"` for below-fold cards
- All images in `/public/images/services/` directory
